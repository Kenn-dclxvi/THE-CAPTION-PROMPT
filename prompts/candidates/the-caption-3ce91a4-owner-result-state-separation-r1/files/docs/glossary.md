# 用語集

## この文書の位置づけ
この文書は、AI 駆動開発プロセスで使う用語の意味を1か所へ集約し、他文書から用語名で参照できるようにする用語集である。

正本は `docs/orchestration-process.md` であり、構成項目の逐語列挙・照合フロー・展開手順・出力様式は正本に置く。この用語集は概念の意味だけを集約し、正本の手続きを複製せず参照に留める。

各用語には、定義の出典(文書名・セクション名)を併記する。

## 指示書
承認済み実装指示書の略語である。
出典: `CLAUDE.md` 冒頭の略語定義。

## SA
サブエージェントの略語である。
出典: `CLAUDE.md` 冒頭の略語定義。

## 親
親エージェントの略語である。親は指示書管理・SA起動・結果管理を行い、実装・修正・テスト・監査・PRレビュー相当の品質確認・指示書作成を直接行わない。親エージェントの希少資源はコンテキストと信頼であり、中核責務はコンテキスト経路設計と関所強制の二つに収束する。
出典: `CLAUDE.md` 冒頭の略語定義・§役割、`docs/orchestration-process.md` §親エージェント。

## 境界項目
対象・範囲・完了条件・品質基準・実行許可を指す。
出典: `CLAUDE.md` 冒頭の用語定義。

## 作業単位化
指示書または明示された作業単位化条件がある場合に限り、依頼を `target / scope / done / tests / stop` に整理して実装SAへ渡すことである。分解に AI側の補完が必要な場合は停止する。
出典: `CLAUDE.md` §作業単位化、`docs/orchestration-process.md` §作業単位化判定フロー。

## 停止指摘
監査SAが使う指摘分類のひとつで、完了条件または品質基準を満たさないため修正が必須なものを指す。
出典: `docs/orchestration-process.md` §指摘分類。

## 任意指摘
監査SAが使う指摘分類のひとつで、今回の完了を妨げない改善を指す。修正は任意である。
出典: `docs/orchestration-process.md` §指摘分類。

## 範囲外指摘
今回の作業単位では扱わない範囲外の指摘を指す。監査SAでは、今回の実装指示書の範囲外であり今回の作業単位では扱わないものを指す。レビューSAでは、承認済み範囲外・仕様拡張・別機能・任意リファクタ・設計変更要求を指す。いずれも自動修正対象にしない。
出典: `docs/orchestration-process.md` §指摘分類。

## 重大指摘
レビューSAが使う指摘分類のひとつで、承認済み範囲内で、マージ前に直すべき明確な欠陥または高リスクを指す。
出典: `docs/orchestration-process.md` §指摘分類。

## 改善指摘
レビューSAが使う指摘分類のひとつで、今回の完了条件を妨げない品質改善を指す。自動修正対象にしない。
出典: `docs/orchestration-process.md` §指摘分類。

## 補足
レビューSAが使う指摘分類のひとつで、修正要求ではない判断メモを指す。自動修正対象にしない。
出典: `docs/orchestration-process.md` §指摘分類。

## 条件付き事前承認テンプレート
明示済みの条件をIDで参照するための定義である。構成項目・照合手順・除外操作の正本は `docs/orchestration-process.md` §条件付き事前承認テンプレートに置く。
出典: `docs/orchestration-process.md` §条件付き事前承認テンプレート・§基本方針、`CLAUDE.md` §入力境界、`docs/prompt-guide.md` §実行許可。

## 既定テスト定義
明示済みのテスト条件をIDで参照するための定義である。構成項目・照合手順・展開手順・運用制約の正本は `docs/orchestration-process.md` §既定テスト定義に置く。
出典: `docs/orchestration-process.md` §既定テスト定義・§基本方針、`docs/prompt-guide.md` §実行許可。
