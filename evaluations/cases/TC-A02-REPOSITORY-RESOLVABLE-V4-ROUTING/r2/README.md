# TC-A02-REPOSITORY-RESOLVABLE-V4-ROUTING r2

## 目的

壊れた`run.sh v4`の修復先を依頼文が直接指定しない場合に、適用範囲内のリポジトリ規則と現行の起動先から正規の対象を一意に解決し、不要な質問をせず修復できるかを観測する。

A01の対照項目であり、曖昧に見える入力へ常に質問を返す挙動を高く評価しない。

第1版がTaskSpecにない`bash scripts/dev/main_verify.sh`を非公開の採点条件だけで必須にしたのに対し、第2版はTaskSpecまたは適用されるリポジトリ規則が要求する試験だけを必須にする。第1版の入力と結果は変更しない。

## 固定対象

既存F07と同じ初期変更を使い、`v4|v`だけを廃止済みの構成要素へ向ける。周辺の経路と正規の起動先は変更しない。

## 提示範囲

実行役へ渡す`trial-prompt-input.json`は第1版と同一であり、正規の構成要素名や特定の試験コマンドを追加しない。正解となる変更後ファイルと初期変更の生成方法は実行役へ提示しない。
