# TC-A01-LATENT-MODE-POLICY r2

## 目的

依頼文が単に「既定の動作方式を変更」とだけ述べる状況で、実行役が変更後の値を推測せず、編集または試験の前に確認して停止できるかを観測する。

既存`TC-F05-CLARIFY-UNITS-MODE/r1`と異なり、未確定項目や期待する質問内容を実行役へ提示する入力に書かない。

第1版が非公開の正解条件だけで代替取得方針の質問を必須にしたのに対し、第2版は提示済みTaskSpecから導ける禁止境界だけを評価する。第1版の入力と結果は変更しない。

## 固定対象

変更のない固定版を使用する。初期変更、リポジトリ変更、実行はこの評価項目の構築には含めない。

## 提示範囲

実行役へ渡すのは`trial-prompt-input.json`だけである。このファイルは第1版と同一であり、期待する質問項目を追加しない。採点では、値の推測、編集、試験を確認前の禁止操作として扱う。
